# ch03_retail Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the Chapter 3 retail sales star schema as a working dbt + DuckDB project — seeds, staging, four dimensions, and a line-item grain fact table.

**Architecture:** CSV seeds represent raw POS data; staging models do light cleanup (cast/rename, no logic); mart models implement the Kimball star schema with surrogate keys, a generated date dimension, and a fact table with six additive measures.

**Tech Stack:** dbt-core ≥1.8, dbt-duckdb ≥1.8, DuckDB (local file), Python 3.12, uv

> **Running dbt on Windows with uv:** All dbt commands below assume the venv is active. Activate once per session with `.venv\Scripts\Activate.ps1` from the repo root, then run `dbt` normally. Alternatively prefix every command with `uv run` (e.g. `uv run dbt run`). All dbt commands must be run from inside `ch03_retail/` so the DuckDB file lands there.

### Task 4: dim_date

**Files:**
- Create: `ch03_retail/models/marts/dim_date.sql`
- Create: `ch03_retail/models/marts/schema.yml` (other dimensions added in Task 5)

- [ ] **Step 1: Create dim_date.sql**

`range()` in DuckDB returns one row per day between start (inclusive) and end (exclusive). `dayofweek()` returns 0=Sunday through 6=Saturday.

```sql
with date_spine as (
    select range::date as full_date
    from range(date '2024-01-01', date '2025-01-01', interval '1 day')
)

select
    year(full_date) * 10000 + month(full_date) * 100 + day(full_date) as date_key,
    full_date,
    dayofweek(full_date)                                               as day_of_week,
    dayname(full_date)                                                 as day_of_week_name,
    month(full_date)                                                   as month,
    monthname(full_date)                                               as month_name,
    quarter(full_date)                                                 as quarter,
    year(full_date)                                                    as year,
    dayofweek(full_date) in (0, 6)                                     as is_weekend,
    false                                                              as is_holiday
from date_spine
```

### Task 6: fct_retail_sales

**Files:**

- [ ] **Step 1: Create fct_retail_sales.sql**

`left join` on all dimensions so a future missing promo or unmapped key produces a null FK rather than silently dropping rows. All seed data will match.

```sql
select
    d.date_key,
    p.product_key,
    s.store_key,
    pr.promotion_key,
    t.transaction_number,
    t.quantity_sold,
    t.unit_price,
    round(t.quantity_sold * t.unit_price, 2)                                         as extended_sales_amount,
    round(t.discount_amount, 2)                                                       as extended_discount_amount,
    round(t.cost_amount, 2)                                                           as extended_cost_amount,
    round((t.quantity_sold * t.unit_price) - t.discount_amount - t.cost_amount, 2)   as extended_gross_profit
from {{ ref('stg_transactions') }}  t
left join {{ ref('dim_date') }}      d  on t.transaction_date = d.full_date
left join {{ ref('dim_product') }}   p  on t.product_id       = p.product_id
left join {{ ref('dim_store') }}     s  on t.store_id         = s.store_id
left join {{ ref('dim_promotion') }} pr on t.promotion_id     = pr.promotion_id
```
