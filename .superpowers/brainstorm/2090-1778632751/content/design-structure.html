What each layer does
Seeds

Raw CSV files representing operational system output — what a real POS system would export. You'll author these to match the Kimball retail scenario.

Staging

One model per source table. Light cleaning only: rename columns to snake_case, cast types, no joins. No Kimball decisions here yet.

Marts

The dimensional model itself. Each model is a deliberate Kimball design choice: surrogate keys, conformed dimensions, grain declaration. This is where the learning happens.
